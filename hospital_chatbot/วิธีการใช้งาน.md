# วิธีการใช้งานระบบแชทบอทโรงพยาบาลมหาวิทยาลัยพะเยา

## ภาพรวมระบบ
ระบบแชทบอทของโรงพยาบาลมหาวิทยาลัยพะเยาเป็นระบบตอบคำถามข้อมูลบริการโรงพยาบาล โดยเน้นการตอบจากฐานข้อมูลความรู้ของโรงพยาบาลเป็นหลัก ไม่ให้โมเดล AI สร้างข้อเท็จจริงเอง

ระบบนี้เหมาะสำหรับ:
- ตารางแพทย์ออกตรวจ
- วัคซีนและบริการผู้ป่วยนอก
- ตรวจสุขภาพและใบรับรองแพทย์
- เบอร์ติดต่อและข้อมูลบริการทั่วไป

ระบบนี้ไม่ใช้แทนแพทย์ ไม่วินิจฉัยโรค และไม่ควรใช้กับข้อมูลผู้ป่วยจริง

## ระบบทำงานร่วมกันอย่างไร
ระบบแบ่งเป็น 3 ชั้น

1. Frontend
เป็นหน้าเว็บและกล่องแชทที่ผู้ใช้เห็น ใช้ Next.js

2. Backend API
ใช้ FastAPI รับคำถามผ่าน `/chat` แล้วจัดหมวด ค้นฐานข้อมูล เลือกคำตอบ ตรวจ policy และส่งคำตอบกลับ

3. Knowledge Base และ Model
ฐานข้อมูลหลักที่ใช้ตอบจริงคือ `data/knowledge.jsonl` และ `data/knowledge.csv`
โมเดล LLM มีหน้าที่ช่วยเรียบเรียงหรือช่วยเข้าใจคำถาม แต่ไม่ใช่แหล่งข้อมูลหลักของโรงพยาบาล

## ไฟล์ฐานข้อมูลหลักที่ต้องเข้าใจ
ไฟล์ต้นฉบับข้อมูลหลักคือ:

```text
data/AIคำถามคำตอบงานสื่อสาร01.04.69.xlsx
```

ไฟล์ที่ระบบใช้ตอนรันจริงเป็นหลักคือ:

```text
data/knowledge.jsonl
data/knowledge.csv
```

รูปตารางแพทย์อยู่ที่:

```text
data/ตารางออกตรวจแพทย์/
```

รูปตรวจสุขภาพอยู่ที่:

```text
data/ตรวจสุขภาพประจำปี/
```

หมายเหตุสำคัญ:
- ไฟล์ `data/ตารางออกตรวจแพทย์/AIคำถามคำตอบงานสื่อสาร01.04.69.xlsx` ไม่ควรถือเป็นฐานข้อมูลหลัก
- โฟลเดอร์ `ตารางออกตรวจแพทย์` ใช้เก็บรูปตารางแพทย์ ไม่ใช่ที่เก็บ Excel หลักของระบบ

## วิธีเปิด Ollama
ใช้สำหรับกรณีรันแบบ local/offline

```bat
set OLLAMA_MODELS=D:\OllamaModels\models
ollama serve
```

ก่อนใช้งานควรตรวจว่า:
- ติดตั้ง Ollama แล้ว
- มีโมเดลที่ระบบจะเรียกใช้
- พอร์ต `11434` พร้อมใช้งาน

## วิธีเปิด Backend
เปิด Command Prompt หรือ PowerShell ใหม่ แล้วรัน:

```bat
cd /d D:\UPH_chatbot\hospital_chatbot
set OLLAMA_MODELS=D:\OllamaModels\models
set LLM_PROVIDER=ollama
set OLLAMA_MODEL=UP_FahMui_GGUF_q4
set OLLAMA_BASE_URL=http://127.0.0.1:11434
set OLLAMA_TIMEOUT_SECONDS=30
set RAG_GROUNDED_LLM=1
set ANSWER_MODE=kb_exact
.venv\Scripts\python.exe -m uvicorn backend.app:app --host 127.0.0.1 --port 8000
```

เมื่อเปิดสำเร็จ Backend จะพร้อมใช้งานที่:

```text
http://127.0.0.1:8000
```

## วิธีเปิด Frontend
สำหรับโหมดพัฒนา:

```bat
cd /d D:\UPH_chatbot\hospital_chatbot\nextjs_frontend
set NEXT_PUBLIC_API_BASE_URL=http://127.0.0.1:8000
set NEXT_PUBLIC_DEBUG_CHAT=0
npm run dev
```

แล้วเปิดเว็บที่:

```text
http://localhost:3000
```

## วิธีเปิด Frontend แบบ Production-like
ใช้สำหรับ UAT หรือเดโมที่เสถียรกว่า `dev`

```bat
cd /d D:\UPH_chatbot\hospital_chatbot\nextjs_frontend
set NEXT_PUBLIC_API_BASE_URL=http://127.0.0.1:8000
set NEXT_PUBLIC_DEBUG_CHAT=0
npm run build
npm run start
```

## วิธีรันทดสอบระบบ
ตรวจ syntax ของไฟล์ backend หลัก:

```bat
cd /d D:\UPH_chatbot\hospital_chatbot
.venv\Scripts\python.exe -m py_compile backend\app.py backend\prompts.py
```

รัน focused regression:

```bat
cd /d D:\UPH_chatbot\hospital_chatbot
python test_focused_runtime_regression.py http://127.0.0.1:8000
```

## วิธีเปิด Cloudflare Tunnel สำหรับเดโม
เปิด Backend และ Frontend ให้เรียบร้อยก่อน แล้วรัน:

```bat
cloudflared tunnel --url http://localhost:3000
```

ระบบจะได้ลิงก์ชั่วคราวลักษณะ:

```text
https://xxxx.trycloudflare.com
```

ข้อควรระวัง:
- ใช้สำหรับเดโมชั่วคราว
- ไม่ควรใช้แทน production deployment
- ห้ามทดสอบด้วยข้อมูลผู้ป่วยจริง

## วิธีอัปเดตฐานความรู้ให้ปลอดภัย
ขั้นตอนที่แนะนำ:

1. แก้ไขข้อมูลต้นฉบับใน `data/AIคำถามคำตอบงานสื่อสาร01.04.69.xlsx`
2. ตรวจว่าข้อมูลเบอร์โทร เวลา และชื่อแผนกถูกต้อง
3. แปลงหรือ rebuild ให้ได้ `data/knowledge.jsonl` และ `data/knowledge.csv` ชุดใหม่
4. รัน regression test ก่อนเปิดใช้งานจริง
5. ให้ทีมโรงพยาบาลทำ UAT ซ้ำในคำถามสำคัญ

ข้อสำคัญ:
- ห้ามแก้ `knowledge.jsonl` แบบสุ่มโดยไม่มีการตรวจสอบต้นทาง
- ให้ถือ Excel หลักเป็นข้อมูลต้นฉบับ และ `knowledge.jsonl` เป็นไฟล์ runtime

## วิธีอัปเดตรูปตารางแพทย์
ถ้าต้องเปลี่ยนรูปแนบ ให้แทนที่ไฟล์ใน:

```text
data/ตารางออกตรวจแพทย์/
```

แนวทางที่แนะนำ:
- ใช้ชื่อไฟล์เดิมเมื่อเป็นการอัปเดตเวอร์ชันล่าสุด
- ตรวจว่ารูปเปิดได้จริงจากหน้าเว็บ
- ตรวจว่าคำตอบที่อ้างอิงรูปแนบยังชี้ไปยังไฟล์ถูกต้อง

## วิธีอัปเดตรูปตรวจสุขภาพ
ให้อัปเดตใน:

```text
data/ตรวจสุขภาพประจำปี/
```

หลังอัปเดตควรทดสอบคำถาม:
- โปรแกรมตรวจสุขภาพ
- ตรวจสุขภาพและใบรับรองแพทย์
- วัคซีนไวรัสตับอักเสบบี

## ปัญหาที่พบบ่อยและวิธีแก้
### เปิดหน้าเว็บได้ แต่แชทไม่ตอบ
- ตรวจว่า Backend เปิดอยู่ที่พอร์ต `8000`
- ตรวจค่า `NEXT_PUBLIC_API_BASE_URL`
- ตรวจว่า frontend เรียก `/chat` ได้จริง

### หน้าเว็บขึ้น แต่รูปแนบไม่แสดง
- ตรวจว่าไฟล์รูปยังอยู่ใน `data/ตารางออกตรวจแพทย์/` หรือ `data/ตรวจสุขภาพประจำปี/`
- ตรวจชื่อไฟล์ว่าตรงกับที่ระบบอ้างอิง

### คำตอบมีข้อความแปลก เช่น path หรือหมายเหตุ
- รัน regression test
- ตรวจว่าใช้โค้ด backend เวอร์ชันล่าสุด
- ให้ทีมเนื้อหาทดสอบด้วยชุดคำถาม UAT ซ้ำ

### Frontend เปิดไม่ได้หลัง build
- ตรวจว่าใช้ Node.js รุ่น LTS
- รัน `npm install` ใหม่
- รัน `npm run build` แล้วดู error log

### Backend เปิดไม่ได้
- ตรวจว่ามี Python 3.11 และ dependency ครบ
- ตรวจว่า `.venv` พร้อมใช้งาน
- ตรวจว่าพอร์ต `8000` ยังไม่ถูกใช้โดย process อื่น

## ข้อควรระวังด้านข้อมูล
- ห้ามใช้ข้อมูลผู้ป่วยจริงในการทดสอบ
- ห้ามส่งเลขบัตรประชาชน เบอร์โทรผู้ป่วย หรือข้อมูลสุขภาพจริงผ่านแชท
- ถ้าเป็นคำถามเชิงวินิจฉัยหรือการรักษา ให้ส่งต่อเจ้าหน้าที่หรือแพทย์
